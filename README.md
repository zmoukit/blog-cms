# Laravel Blog & CMS

## Overview
This is a **Laravel Blog & CMS project** built to showcase full-stack Laravel skills, including user authentication, role-based access, RESTful APIs, and content management.  
The project demonstrates best practices in **MVC structure, API development, testing, and deployment**.
The data model is designed using clear domain relationships with **Eloquent ORM**, following best practices for maintainability and readability.

## Purpose
The main goals of this project are:
- To practice building a full-featured Laravel application from scratch
- To showcase **role-based user access**, **API development**, and **testing practices**

## Features
- **User Authentication**: Admin, Author, Reader roles
- **CRUD Operations**: Posts, Categories, Comments
- **REST API**: Endpoints for posts and comments
- **Testing**: Unit tests for models, feature tests for APIs
- **Deployment**: Ready for deployment on Laravel Forge or Railway

## Technologies Used
- PHP 8.x & Laravel 11
- PostgreSQL
- Laravel Sanctum for API authentication
- Laravel Breeze for authentication scaffolding
- Blade && TailwindCSS
- Testing: PHPUnit & Laravel Feature/Unit Tests
